ANU Attendance System

Accessibility Policy

Africa Nazarene University is committed to making this attendance system accessible and usable for everyone, regardless of ability or technology.

Last updated: March 2026

♿ Our Commitment

The ANU Student Sign-In System is designed to be accessible to all users, including those with visual, auditory, motor, or cognitive disabilities. We follow internationally recognised accessibility standards and continuously work to improve the experience for every user.

This policy describes the specific measures we have implemented across the system to ensure accessibility. We welcome feedback from users to help us identify and address any accessibility barriers.

✓ WCAG 2.1 AA Aligned ✓ Screen Reader Friendly ✓ Keyboard Accessible

🛠 Accessibility Features

🖼

Alternative Text for Images

All images on this website include descriptive alt attributes, enabling screen readers to convey visual information to users who are blind or have low vision. Decorative images are marked with aria-hidden="true" to avoid unnecessary clutter.

⌨️

Keyboard Navigation

Every interactive element — forms, buttons, navigation links, tabs — can be accessed and operated using only a keyboard. Focus indicators are clearly visible so users always know where they are on the page. Pressing Tab moves forward, Shift+Tab moves backward, and Enter or Space activates controls.

⏭

Skip Links

A "Skip to main content" link is included at the top of every page. This allows keyboard and screen reader users to bypass the navigation menu and jump directly to the main content, avoiding repetitive navigation on every page load.

🎨

Colour Contrast

Text and interactive elements meet WCAG 2.1 AA contrast ratio requirements (minimum 4.5:1 for normal text). Our navy and gold colour scheme was specifically chosen for strong contrast. Dark mode is also available for users who prefer reduced brightness or have photosensitivity.

📝

Forms and Labels

All form fields have explicit <label> elements linked via for and id attributes. Required fields are marked both visually and with aria-required="true". Error messages use role="alert" and aria-live="polite" so assistive technologies announce them automatically.

📐

Clear and Consistent Structure

Pages use a logical heading hierarchy (H1 → H2 → H3) and semantic HTML elements such as <nav>, <main>, <section>, <header>, and <footer>. Landmark roles help screen reader users navigate by jumping between page regions.

🔤

Resizable Text

All text is defined using relative units (rem, em), so users can increase their browser's default font size up to 200% without loss of content or functionality. The layout adapts responsively to accommodate larger text sizes.

📱

Responsive Design

The system is fully responsive across devices — desktop, tablet, and mobile — ensuring all users can access the sign-in system regardless of the device or screen size they are using.

🌙

Dark Mode

A dark mode toggle is available in the navigation bar. This benefits users with photosensitivity, migraines, or visual fatigue, and improves usability in low-light environments. The user's preference is saved automatically.

⚠️ Known Limitations

While we strive for full accessibility, the following areas are still being improved:

Video captioning — this system does not currently include video content, so captioning is not applicable. If video content is added in future, captions will be provided.

Barcode and QR code images — generated codes include a text token below them for screen reader users. We continue to explore better ways to make these outputs more accessible.

💬 Feedback and Contact

If you experience any accessibility barrier on this site, or if you need content in an alternative format, please contact us. We aim to respond to accessibility queries within 5 working days.
